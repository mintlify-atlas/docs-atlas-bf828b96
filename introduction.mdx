Overload API is a backend service for strength training applications. It goes beyond simple workout logging — it automatically calculates training volume, detects new personal records the moment they occur, and estimates one-rep max so athletes can plan their training loads with data.

Built with **NestJS 11**, **PostgreSQL**, and **Prisma**, the API exposes a fully authenticated REST interface with JWT + refresh token auth, modular architecture, and Docker support.

## Key capabilities

| Feature | Description |
|---|---|
| JWT authentication | Stateless access tokens (15 min) + rotating refresh tokens (7 days) |
| Exercise catalog | Personal library of exercises with soft-delete archiving |
| Routines | Workout templates with ordered exercises and target parameters |
| Workout sessions | Real-time tracking, one active session enforced per user |
| Set logging | Weight, reps, RPE, warmup flag per set |
| PR detection | Weight PR and volume PR per exercise, calculated on demand |
| 1RM estimation | Epley formula applied to non-warmup sets with ≤ 10 reps |
| Volume tracking | Per-set and per-workout volume (weight × reps) |
| Progression history | Session-by-session exercise progression over time |

http://localhost:3000
