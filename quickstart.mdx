This guide walks you through starting the server, creating an account, and making your first authenticated requests against the Overload API.

## Prerequisites

- [Docker](https://www.docker.com/) and Docker Compose installed
- `curl` available in your terminal

<Steps>
  <Step title="Start the server">
    Clone the repository and copy the environment file:

    ```bash
    git clone https://github.com/JosepRivera/overload-api.git
    cd overload-api
    cp .env.example .env
    ```

    Open `.env` and fill in `POSTGRES_PASSWORD`, `JWT_SECRET`, and any other required values. Then start the development environment:

    ```bash
    pnpm dev
    ```

    This starts two containers — a PostgreSQL database and the NestJS app — with hot-reload enabled. Pending migrations are applied automatically on startup.

    <Note>
      The API listens on `http://localhost:3000` by default. Interactive Swagger docs are available at `http://localhost:3000/api/docs`.
    </Note>
  </Step>

  <Step title="Register an account">
    Send a `POST /auth/register` request with your email, display name, and password. On success, the API returns an access token and a refresh token — you are logged in immediately.

    ```bash
    curl -s -X POST http://localhost:3000/auth/register \
      -H "Content-Type: application/json" \
      -d '{
        "email": "alex@example.com",
        "name": "Alex",
        "password": "mysecretpassword"
      }'
    ```

    **Response `201 Created`:**

    ```json
    {
      "data": {
        "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
        "refreshToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
        "user": {
          "id": "cm9abc123def456ghi789",
          "email": "alex@example.com",
          "name": "Alex",
          "is_active": true,
          "created_at": "2026-04-03T10:00:00.000Z",
          "updated_at": "2026-04-03T10:00:00.000Z"
        }
      }
    }
    ```

    Copy the `accessToken` value — you will use it in the next step.

    <Tip>
      Store the `refreshToken` securely. You will need it to get a new access token when the current one expires after 15 minutes.
    </Tip>
  </Step>

  <Step title="Make an authenticated request">
    Pass the access token as a `Bearer` token in the `Authorization` header. The following request fetches your personal exercise catalog:

    ```bash
    curl -s http://localhost:3000/exercises \
      -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
    ```

    **Response `200 OK`:**

    ```json
    {
      "data": []
    }
    ```

    The catalog is empty for a new account. Create your first exercise before logging sets:

    ```bash
    curl -s -X POST http://localhost:3000/exercises \
      -H "Content-Type: application/json" \
      -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..." \
      -d '{
        "name": "Bench Press",
        "category": "chest",
        "type": "compound"
      }'
    ```

    **Response `201 Created`:**

    ```json
    {
      "data": {
        "id": "cm9xyz789abc123def456",
        "name": "Bench Press",
        "category": "chest",
        "type": "compound",
        "is_archived": false,
        "created_at": "2026-04-03T10:01:00.000Z",
        "updated_at": "2026-04-03T10:01:00.000Z"
      }
    }
    ```
  </Step>

  <Step title="Log your first set">
    Start a workout session and log a set against the exercise you just created.

    **Start a workout:**

    ```bash
    curl -s -X POST http://localhost:3000/workouts \
      -H "Content-Type: application/json" \
      -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..." \
      -d '{
        "started_at": "2026-04-03T10:05:00.000Z"
      }'
    ```

    **Log a set** using the `workout_id` from the response above and the `exercise_id` from step 3:

    ```bash
    curl -s -X POST http://localhost:3000/workouts/cm9wkt001abc/sets \
      -H "Content-Type: application/json" \
      -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..." \
      -d '{
        "exercise_id": "cm9xyz789abc123def456",
        "set_number": 1,
        "weight": 80,
        "reps": 8
      }'
    ```

    When you are done training, finish the session by calling `POST /workouts/:id/finish`. Once finished, the workout and its sets become immutable.

    <Note>
      Volume (`weight × reps`), personal records, and 1RM estimates are calculated on demand. Query the analytics endpoints to see your stats after each session.
    </Note>

    See the [sets API reference](/api-reference/workouts/sets) for all available set fields, including `rpe` (Rate of Perceived Exertion) and `is_warmup`.
  </Step>
</Steps>
