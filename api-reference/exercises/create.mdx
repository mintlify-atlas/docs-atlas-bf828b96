---
title: 'Create exercise'
description: 'Create a new exercise for the authenticated user. Exercise names must be unique per user.'
api: 'POST /exercises'
---

<Note>
  This endpoint requires a valid access token. Pass it as `Authorization: Bearer <token>` in the request header.
</Note>

## Request body

<ParamField body="name" type="string" required>
  The exercise name. Must be between 1 and 150 characters. Leading and trailing whitespace is trimmed. Must be unique per user â€” you cannot have two exercises with the same name.
</ParamField>

<ParamField body="category" type="string" required>
  The muscle group or training category. Must be one of: `chest`, `back`, `legs`, `shoulders`, `arms`, `core`, `cardio`, `other`.
</ParamField>

<ParamField body="type" type="string" required>
  The movement type. Must be one of: `compound`, `isolation`, `cardio`, `stretching`.
</ParamField>

<ParamField body="notes" type="string">
  Optional free-text notes about the exercise, such as form cues or equipment. Maximum 2000 characters.
</ParamField>

## Response

<ResponseField name="id" type="string" required>
  The exercise's unique identifier (UUID).
</ResponseField>

<ResponseField name="user_id" type="string" required>
  The UUID of the user who owns this exercise.
</ResponseField>

<ResponseField name="name" type="string" required>
  The exercise name.
</ResponseField>

<ResponseField name="category" type="string" required>
  The exercise category.
</ResponseField>

<ResponseField name="type" type="string" required>
  The movement type.
</ResponseField>

<ResponseField name="notes" type="string">
  Free-text notes, or `null` if none were provided.
</ResponseField>

<ResponseField name="is_archived" type="boolean" required>
  Whether the exercise has been archived. Always `false` for newly created exercises.
</ResponseField>

<ResponseField name="created_at" type="string" required>
  ISO 8601 timestamp of when the exercise was created.
</ResponseField>

<ResponseField name="updated_at" type="string" required>
  ISO 8601 timestamp of the last update.
</ResponseField>

## Example

```bash
curl --request POST \
  --url http://localhost:3000/exercises \
  --header 'Authorization: Bearer <token>' \
  --header 'Content-Type: application/json' \
  --data '{
    "name": "Barbell Back Squat",
    "category": "legs",
    "type": "compound",
    "notes": "Keep chest up, drive knees out."
  }'
```

## Error responses

| Status | Error | Description |
|--------|-------|-------------|
| `401` | `Unauthorized` | Missing or invalid access token. |
| `409` | `Conflict` | An exercise with this name already exists for your account. |
