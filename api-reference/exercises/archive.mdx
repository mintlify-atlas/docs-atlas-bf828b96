---
title: 'Archive exercise'
description: 'Soft-delete an exercise by marking it as archived. All historical set data is preserved.'
api: 'PATCH /exercises/{id}/archive'
---

<Note>
  This endpoint requires a valid access token. Pass it as `Authorization: Bearer <token>` in the request header.
</Note>

<Warning>
  Archiving is a soft delete — it sets `is_archived` to `true` and hides the exercise from the default list response. All workout sets logged against this exercise are preserved and remain accessible in analytics and workout history. This action cannot be undone through the API.
</Warning>

## Path parameters

<ParamField path="id" type="string" required>
  The UUID of the exercise to archive. Must be a valid UUID v4.
</ParamField>

## Response

Returns the updated exercise object with `is_archived` set to `true`.

<ResponseField name="id" type="string" required>
  The exercise's unique identifier (UUID).
</ResponseField>

<ResponseField name="user_id" type="string" required>
  The UUID of the user who owns this exercise.
</ResponseField>

<ResponseField name="name" type="string" required>
  The exercise name.
</ResponseField>

<ResponseField name="category" type="string" required>
  The exercise category.
</ResponseField>

<ResponseField name="type" type="string" required>
  The movement type.
</ResponseField>

<ResponseField name="notes" type="string">
  Free-text notes, or `null` if none are set.
</ResponseField>

<ResponseField name="is_archived" type="boolean" required>
  Always `true` after a successful archive request.
</ResponseField>

<ResponseField name="created_at" type="string" required>
  ISO 8601 timestamp of when the exercise was created.
</ResponseField>

<ResponseField name="updated_at" type="string" required>
  ISO 8601 timestamp of the archive operation.
</ResponseField>

## Example

```bash
curl --request PATCH \
  --url http://localhost:3000/exercises/018e8f2a-1b3c-7d4e-9f0a-2b3c4d5e6f7a/archive \
  --header 'Authorization: Bearer <token>'
```

```json
{
  "data": {
    "id": "018e8f2a-1b3c-7d4e-9f0a-2b3c4d5e6f7a",
    "user_id": "018e1a2b-3c4d-5e6f-7a8b-9c0d1e2f3a4b",
    "name": "Barbell Back Squat",
    "category": "legs",
    "type": "compound",
    "notes": "Keep chest up, drive knees out.",
    "is_archived": true,
    "created_at": "2026-04-03T10:00:00.000Z",
    "updated_at": "2026-04-03T12:00:00.000Z"
  }
}
```

## Error responses

| Status | Error | Description |
|--------|-------|-------------|
| `401` | `Unauthorized` | Missing or invalid access token. |
| `404` | `Not Found` | No exercise with the given ID exists for your account. |
