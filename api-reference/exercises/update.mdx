---
title: 'Update exercise'
description: 'Update one or more fields on an existing exercise. All body fields are optional — only the fields you include are changed.'
api: 'PATCH /exercises/{id}'
---

<Note>
  This endpoint requires a valid access token. Pass it as `Authorization: Bearer <token>` in the request header.
</Note>

## Path parameters

<ParamField path="id" type="string" required>
  The UUID of the exercise to update. Must be a valid UUID v4.
</ParamField>

## Request body

All fields are optional. Only the fields you include are updated.

<ParamField body="name" type="string">
  A new name for the exercise. Must be between 1 and 150 characters. Leading and trailing whitespace is trimmed. Must be unique per user — you cannot rename to a name already used by another of your exercises.
</ParamField>

<ParamField body="category" type="string">
  A new category. Must be one of: `chest`, `back`, `legs`, `shoulders`, `arms`, `core`, `cardio`, `other`.
</ParamField>

<ParamField body="type" type="string">
  A new movement type. Must be one of: `compound`, `isolation`, `cardio`, `stretching`.
</ParamField>

<ParamField body="notes" type="string">
  Updated free-text notes. Maximum 2000 characters. Pass `null` to clear existing notes.
</ParamField>

## Response

Returns the full updated exercise object.

<ResponseField name="id" type="string" required>
  The exercise's unique identifier (UUID).
</ResponseField>

<ResponseField name="user_id" type="string" required>
  The UUID of the user who owns this exercise.
</ResponseField>

<ResponseField name="name" type="string" required>
  The exercise name.
</ResponseField>

<ResponseField name="category" type="string" required>
  The exercise category.
</ResponseField>

<ResponseField name="type" type="string" required>
  The movement type.
</ResponseField>

<ResponseField name="notes" type="string">
  Free-text notes, or `null` if none are set.
</ResponseField>

<ResponseField name="is_archived" type="boolean" required>
  Whether the exercise has been archived.
</ResponseField>

<ResponseField name="created_at" type="string" required>
  ISO 8601 timestamp of when the exercise was created.
</ResponseField>

<ResponseField name="updated_at" type="string" required>
  ISO 8601 timestamp of the last update. Reflects the current update.
</ResponseField>

## Example

```bash
curl --request PATCH \
  --url http://localhost:3000/exercises/018e8f2a-1b3c-7d4e-9f0a-2b3c4d5e6f7a \
  --header 'Authorization: Bearer <token>' \
  --header 'Content-Type: application/json' \
  --data '{
    "name": "Low Bar Back Squat",
    "notes": "Bar sits lower on rear delts. Wider stance, more forward lean."
  }'
```

## Error responses

| Status | Error | Description |
|--------|-------|-------------|
| `401` | `Unauthorized` | Missing or invalid access token. |
| `404` | `Not Found` | No exercise with the given ID exists for your account. |
| `409` | `Conflict` | Another exercise with the requested name already exists for your account. |
