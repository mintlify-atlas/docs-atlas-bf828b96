---
title: 'Get & delete a workout'
description: 'Retrieve a single workout by ID, or delete a workout that has no sets.'
---

<Note>
  All endpoints on this page require a valid access token. Pass it as `Authorization: Bearer <token>` in the request header.
</Note>

## GET /workouts/:id

Return a single workout by its ID, including all logged sets.

### Path parameters

<ParamField path="id" type="string" required>
  The UUID of the workout to retrieve. Must be a valid UUID v4.
</ParamField>

### Response

Returns the workout object with its associated sets.

<ResponseField name="id" type="string" required>
  The workout's unique identifier (UUID).
</ResponseField>

<ResponseField name="user_id" type="string" required>
  The UUID of the user who owns this workout.
</ResponseField>

<ResponseField name="routine_id" type="string">
  The UUID of the associated routine, or `null` for a free workout.
</ResponseField>

<ResponseField name="started_at" type="string" required>
  ISO 8601 timestamp of when the session began.
</ResponseField>

<ResponseField name="finished_at" type="string">
  ISO 8601 timestamp of when the session was completed, or `null` if the workout is still active.
</ResponseField>

<ResponseField name="notes" type="string">
  Free-text notes, or `null` if none were provided.
</ResponseField>

<ResponseField name="created_at" type="string" required>
  ISO 8601 timestamp of when the record was created.
</ResponseField>

<Note>
  To retrieve the sets for a workout, use [GET /workouts/:workoutId/sets](/api-reference/workouts/sets).
</Note>

### Example

```bash
curl --request GET \
  --url http://localhost:3000/workouts/b2c3d4e5-f6a7-8b9c-0d1e-2f3a4b5c6d7e \
  --header 'Authorization: Bearer <token>'
```

```json
{
  "data": {
    "id": "b2c3d4e5-f6a7-8b9c-0d1e-2f3a4b5c6d7e",
    "user_id": "018e1a2b-3c4d-5e6f-7a8b-9c0d1e2f3a4b",
    "routine_id": "a1b2c3d4-e5f6-7a8b-9c0d-1e2f3a4b5c6d",
    "started_at": "2026-04-03T09:00:00.000Z",
    "finished_at": "2026-04-03T10:15:00.000Z",
    "notes": "Felt strong today.",
    "created_at": "2026-04-03T09:00:01.000Z"
  }
}
```

### Error responses

| Status | Error | Description |
|--------|-------|-------------|
| `401` | `Unauthorized` | Missing or invalid access token. |
| `404` | `Not Found` | No workout with the given ID exists for your account. |

---

## DELETE /workouts/:id

Delete a workout. Only workouts with no logged sets can be deleted. If the workout has sets, you must delete all sets first, or [finish the workout](/api-reference/workouts/finish) instead.

<Warning>
  This operation permanently deletes the workout record. It cannot be undone.
</Warning>

### Path parameters

<ParamField path="id" type="string" required>
  The UUID of the workout to delete. Must be a valid UUID v4.
</ParamField>

### Response

Returns `204 No Content` on success. The response body is empty.

### Example

```bash
curl --request DELETE \
  --url http://localhost:3000/workouts/b2c3d4e5-f6a7-8b9c-0d1e-2f3a4b5c6d7e \
  --header 'Authorization: Bearer <token>'
```

### Error responses

| Status | Error | Description |
|--------|-------|-------------|
| `401` | `Unauthorized` | Missing or invalid access token. |
| `404` | `Not Found` | No workout with the given ID exists for your account. |
| `409` | `Conflict` | The workout has associated sets and cannot be deleted. Remove all sets first. |
