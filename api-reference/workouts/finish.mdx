---
title: 'Finish a workout'
description: 'Mark an active workout session as completed.'
api: 'POST /workouts/{id}/finish'
---

<Note>
  This endpoint requires a valid access token. Pass it as `Authorization: Bearer <token>` in the request header.
</Note>

Mark an active workout as completed. The server sets `finished_at` to the current timestamp (`NOW()`). Once finished, a workout's sets become immutable — they can no longer be added, modified, or deleted.

<Warning>
  Finishing a workout is irreversible. You cannot reopen a finished workout. If you made a mistake, you can still update the workout's `notes` field after finishing, but sets cannot be changed.
</Warning>

## Path parameters

<ParamField path="id" type="string" required>
  The UUID of the workout to finish. Must be a valid UUID v4.
</ParamField>

## Response

Returns `200 OK` with the updated workout object.

<ResponseField name="id" type="string" required>
  The workout's unique identifier (UUID).
</ResponseField>

<ResponseField name="user_id" type="string" required>
  The UUID of the user who owns this workout.
</ResponseField>

<ResponseField name="routine_id" type="string">
  The UUID of the associated routine, or `null` for a free workout.
</ResponseField>

<ResponseField name="started_at" type="string" required>
  ISO 8601 timestamp of when the session began.
</ResponseField>

<ResponseField name="finished_at" type="string" required>
  ISO 8601 timestamp set by the server at the moment the workout was finished.
</ResponseField>

<ResponseField name="notes" type="string">
  Free-text notes, or `null` if none were provided.
</ResponseField>

<ResponseField name="created_at" type="string" required>
  ISO 8601 timestamp of when the record was created.
</ResponseField>

## What changes after finishing

| Action | Active workout | Finished workout |
|--------|---------------|-----------------|
| Add sets | Allowed | Not allowed |
| Update sets | Allowed | Not allowed |
| Delete sets | Allowed | Not allowed |
| Update `notes` | Allowed | Allowed |
| Delete workout | Allowed (if no sets) | Not allowed |

## Example

```bash
curl --request POST \
  --url http://localhost:3000/workouts/b2c3d4e5-f6a7-8b9c-0d1e-2f3a4b5c6d7e/finish \
  --header 'Authorization: Bearer <token>'
```

```json
{
  "data": {
    "id": "b2c3d4e5-f6a7-8b9c-0d1e-2f3a4b5c6d7e",
    "user_id": "018e1a2b-3c4d-5e6f-7a8b-9c0d1e2f3a4b",
    "routine_id": "a1b2c3d4-e5f6-7a8b-9c0d-1e2f3a4b5c6d",
    "started_at": "2026-04-03T09:00:00.000Z",
    "finished_at": "2026-04-03T10:15:32.000Z",
    "notes": null,
    "created_at": "2026-04-03T09:00:01.000Z"
  }
}
```

## Error responses

| Status | Error | Description |
|--------|-------|-------------|
| `401` | `Unauthorized` | Missing or invalid access token. |
| `404` | `Not Found` | No workout with the given ID exists for your account. |
| `400` | `Bad Request` | The workout is already finished, or the session exceeds 6 hours duration. |
