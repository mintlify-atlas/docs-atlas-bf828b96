---
title: 'Start a workout'
description: 'Start a new workout session and get the current active session.'
---

<Note>
  All endpoints on this page require a valid access token. Pass it as `Authorization: Bearer <token>` in the request header.
</Note>

## POST /workouts

Start a new workout session. Only one active workout is allowed at a time — you must finish or delete your current session before starting a new one.

### Request body

<ParamField body="started_at" type="string" required>
  The time the session began, as an ISO 8601 datetime string (e.g. `2026-04-03T09:00:00.000Z`). Cannot be in the future.
</ParamField>

<ParamField body="routine_id" type="string">
  The UUID of the routine to associate with this workout. Optional — omit to start a free workout with no template.
</ParamField>

<ParamField body="notes" type="string">
  Free-text notes about this workout session. Maximum 2000 characters. Pass `null` to leave empty.
</ParamField>

### Response

Returns `201 Created` with the new workout object.

<ResponseField name="id" type="string" required>
  The workout's unique identifier (UUID).
</ResponseField>

<ResponseField name="user_id" type="string" required>
  The UUID of the user who owns this workout.
</ResponseField>

<ResponseField name="routine_id" type="string">
  The UUID of the associated routine, or `null` for a free workout.
</ResponseField>

<ResponseField name="started_at" type="string" required>
  ISO 8601 timestamp of when the session began.
</ResponseField>

<ResponseField name="finished_at" type="string">
  ISO 8601 timestamp of when the session ended. `null` for an active (in-progress) workout.
</ResponseField>

<ResponseField name="notes" type="string">
  Free-text notes, or `null` if none were provided.
</ResponseField>

<ResponseField name="created_at" type="string" required>
  ISO 8601 timestamp of when the record was created.
</ResponseField>

### Example

```bash
curl --request POST \
  --url http://localhost:3000/workouts \
  --header 'Authorization: Bearer <token>' \
  --header 'Content-Type: application/json' \
  --data '{
    "started_at": "2026-04-03T09:00:00.000Z",
    "routine_id": "a1b2c3d4-e5f6-7a8b-9c0d-1e2f3a4b5c6d"
  }'
```

```json
{
  "data": {
    "id": "b2c3d4e5-f6a7-8b9c-0d1e-2f3a4b5c6d7e",
    "user_id": "018e1a2b-3c4d-5e6f-7a8b-9c0d1e2f3a4b",
    "routine_id": "a1b2c3d4-e5f6-7a8b-9c0d-1e2f3a4b5c6d",
    "started_at": "2026-04-03T09:00:00.000Z",
    "finished_at": null,
    "notes": null,
    "created_at": "2026-04-03T09:00:01.000Z"
  }
}
```

### Error responses

| Status | Error | Description |
|--------|-------|-------------|
| `401` | `Unauthorized` | Missing or invalid access token. |
| `409` | `Conflict` | You already have an active workout. Finish or delete it before starting a new one. |

## GET /workouts/active

Get the current active (in-progress) workout session. Returns `null` if there is no active workout.

<Warning>
  This endpoint must be called before `GET /workouts/:id`. The `/active` path is matched before the `/:id` wildcard, so it will never be treated as an ID lookup.
</Warning>

### Response

Returns the active workout object, or `null` if no workout is currently in progress.

<ResponseField name="id" type="string" required>
  The workout's unique identifier (UUID).
</ResponseField>

<ResponseField name="user_id" type="string" required>
  The UUID of the user who owns this workout.
</ResponseField>

<ResponseField name="routine_id" type="string">
  The UUID of the associated routine, or `null` for a free workout.
</ResponseField>

<ResponseField name="started_at" type="string" required>
  ISO 8601 timestamp of when the session began.
</ResponseField>

<ResponseField name="finished_at" type="string">
  Always `null` — an active workout has not yet been finished.
</ResponseField>

<ResponseField name="notes" type="string">
  Free-text notes, or `null` if none were provided.
</ResponseField>

<ResponseField name="created_at" type="string" required>
  ISO 8601 timestamp of when the record was created.
</ResponseField>

### Example

```bash
curl --request GET \
  --url http://localhost:3000/workouts/active \
  --header 'Authorization: Bearer <token>'
```

```json
{
  "data": {
    "id": "b2c3d4e5-f6a7-8b9c-0d1e-2f3a4b5c6d7e",
    "user_id": "018e1a2b-3c4d-5e6f-7a8b-9c0d1e2f3a4b",
    "routine_id": "a1b2c3d4-e5f6-7a8b-9c0d-1e2f3a4b5c6d",
    "started_at": "2026-04-03T09:00:00.000Z",
    "finished_at": null,
    "notes": null,
    "created_at": "2026-04-03T09:00:01.000Z"
  }
}
```

### Error responses

| Status | Error | Description |
|--------|-------|-------------|
| `401` | `Unauthorized` | Missing or invalid access token. |
